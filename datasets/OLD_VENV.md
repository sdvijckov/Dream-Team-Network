# 📋 Старое окружение dog-wisdom-datasets-venv

В корне проекта есть папка `dog-wisdom-datasets-venv\` со старым виртуальным окружением.

## Что делать?

### Вариант 1: Использовать старое окружение ✅

Если в нём уже установлены нужные зависимости:

```bash
# Активировать старое окружение
cd "D:\Projects python\Dream Team Network"
dog-wisdom-datasets-venv\Scripts\activate

# Перейти в datasets
cd datasets

# Проверить зависимости
pip list

# Если всё ок - можно работать
python cli.py info
```

**Плюсы:**
- Не нужно переустанавливать зависимости
- Экономия времени и места

**Минусы:**
- Могут быть конфликты версий
- Непонятно, что там установлено

---

### Вариант 2: Создать новое окружение ✅ (рекомендуется)

```bash
cd "D:\Projects python\Dream Team Network\datasets"
install.bat
```

**Плюсы:**
- Чистое окружение только для datasets
- Все зависимости известны (requirements.txt)
- Лёгко воспроизвести

**Минусы:**
- Займёт ~500MB места
- Нужно время на установку

---

### Вариант 3: Удалить старое

Если новое окружение работает:

```bash
# Удалить папку старого окружения
rmdir /s /q dog-wisdom-datasets-venv
```

---

## Рекомендация

**Используй новое окружение в `datasets\venv\`** — это чище и правильнее с точки зрения структуры проекта.

Старое окружение можно:
- Переименовать в `dog-wisdom-datasets-venv-backup`
- Протестировать работу
- Удалить если всё ок

---

## Проверка работы

После установки проверь:

```bash
cd "D:\Projects python\Dream Team Network\datasets"
venv\Scripts\activate
python cli.py info
```

Если показывает информацию о системе — всё работает! ✅
